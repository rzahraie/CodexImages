Implement JSON export for the new automated NT8 container construction indicator.

Goal:
Create a machine-readable container ledger so the container engine can be analyzed externally and later used for Gaussian construction.

Do not change container construction logic in this task unless required to expose/export existing state.

Add JSON export only.

---

## Output file

Export one JSON file per chart/session.

Recommended filename:

APVA_Containers_<Instrument>*<BarsPeriod>*<StartBar>*<EndBar>*<Timestamp>.json

Recommended folder:

C:\Users\rz0\Documents\ApvaAnalysis\ContainerJSON\

Expose as user-configurable parameter:

ExportJson = true / false
JsonExportFolder
JsonFileName optional

---

4. barMembership

---

This is critical.

Every processed bar should have a membership record showing all containers that include that bar.

Fields:

barIndex
containerIds
primaryContainerId
roles

roles should be an array:

{
"containerId": 37,
"role": "P3"
}

Possible roles:

Member
P1
P2
P3
FTT
RTLTouch
LTLTouch
VE
Break
Start
End
Ambiguous

Example:

{
"barIndex": 254,
"containerIds": [37, 12, 3],
"primaryContainerId": 37,
"roles": [
{ "containerId": 37, "role": "P3" },
{ "containerId": 12, "role": "Member" },
{ "containerId": 3, "role": "Member" }
]
}

A bar may have multiple roles in multiple containers.

## Validation before export

Before writing JSON, run conservative validation:

* container startBar <= endBar
* p1/p2/p3/ftt ordering if anchors exist
* parent-child references reciprocal
* no parent-child cycles
* child level greater than parent level
* child range should be inside parent range unless explicitly marked as structurally justified
* every container should appear in membership for its start/end bars
* line style should match level

Do not crash NinjaTrader on validation failure.

Add warning records instead.

## Export timing

Export when:

* historical processing completes, or
* user clicks reload / indicator refresh completes

Avoid writing the file on every bar unless explicitly enabled.

Add parameter:

ExportOnEveryBar = false

Default false.

## Important

Do not only export final drawn lines.

Export:

* final container state
* bar membership
* construction events
* validation warnings

This export will be used to evaluate whether the three-level container hierarchy is good enough for Gaussian construction.

Deliverable:
Return updated indicator source file(s) and a short summary of changes.
